\section{Introduction}
Sudoku seems simple enough---fill an $n \times n$ grid so no row, column, or subgrid repeats a digit---but the underlying constraint satisfaction problem scales poorly. Larger grids get hard fast, and that difficulty has made Sudoku a popular testbed in optimization research~\cite{lloyd2020,simonis2005sudoku,manyam2024,SSAS}. Traditional approaches like backtracking~\cite{norvig2006solving,Schott,bhattarai2025study,is2012techniques} and pencil-and-paper elimination~\cite{crook2009pencil,bhattarai2025study} can handle $9 \times 9$ boards without much trouble, but they tend to fall apart on bigger instances. Swarm-based metaheuristics stepped in to address this: Artificial Bee Colony~\cite{pacurib2009}, Particle Swarm Optimization~\cite{PSO,GPSO}, and Ant Colony Optimization (ACO)~\cite{lloyd2020} have all been applied, with ACO showing particular promise when combined with constraint propagation (CP) to weed out dead-end assignments~\cite{norvig2006solving,lloyd2020}.

A few ACO-based solvers deserve a closer look. Lloyd and Amos~\cite{lloyd2020} pair CP with ACS~\cite{dorigo2004ant} and introduce a Best Value Evaporation operator that helps avoid pheromone stagnation. The downside is that their single-colony design sheds diversity on harder puzzles. Mo et al.~\cite{mo2022} tackle diversity more directly with DCM-ACO, running two ACS and one MMAS~\cite{dorigo2004ant} sub-colony and coordinating them through entropy-based game theory, pheromone fusion, and public path recommendation. It works, but the whole thing is single-threaded, so there is no real way to scale it up. Yang and Fang~\cite{yang2015} go in the other direction with RMACO, spreading homogeneous colonies across MPI processors using ring and random-matching communication topologies with an adaptive exchange schedule. The trade-off there is that every colony behaves the same way---no role differentiation, no internal diversity control. Earlier parallel ACO designs~\cite{randall2002} run into their own problems, mostly redundant exploration and synchronization overhead.

So each of these approaches gets part of the picture right but leaves something out. Nobody has combined constraint-aware search, heterogeneous multi-colony cooperation, and structured parallel communication in a single solver. We built \textbf{SudoSLVRR} to do exactly that. It wraps DCM-ACO in a multithreaded dual-topology architecture: each thread manages its own set of heterogeneous colonies, with entropy-regulated cooperation handling coordination within a thread and ring plus random-matching exchanges handling coordination across threads. We tested the framework on $9 \times 9$, $16 \times 16$, and $25 \times 25$ puzzles, and it holds up well on both convergence and diversity.
