To evaluate the performance of SudoSLVRR (MCD), it was compared with the baseline solvers namely Constraint Propagation and Ant Colony System and Constraint Propagation and Multi-Colony ant Optimization with Dynamic Collaborative Mechanism (CD) using four performance metrics. A total of 45 puzzles, 15 instances each for $9\times9$, $16\times16$, and $25\times25$ Sudoku puzzle, were used and can be downloaded here: {link}. Different sizes of Sudoku puzzles are used specifically $9\times9$, $16\times16$, and $25\times25$ with 100 runs per instance in each puzzle size to accurately measure the success rate, average completion time, and average number of iterations. 

From the experimental results, we can see that SudoSLVRR (MCD) was able to achieve a higher success rate compared to CA and CD solvers. Although CP-DCM-ACO (CD) solver has  heterogeneous colonies that balances exploration and exploitation, it still wasn’t able to find a solution when solving high-order Sudoku puzzles. This limitation shows that a further exploration mechanism is needed when solving high-order Sudoku puzzle as the solution space grows larger when the puzzle size increases. 

The parallel search allows the SudoSLVRR (MCD) to cover more regions in the solution space thus achieving a higher success rate than the single-thread solvers. The communication between threads also helps the threads not to visit the same region by sharing pheromone information thus speeding up the search for a solution.
